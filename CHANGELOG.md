# Changelog

## v1.19.0 (2026-06-27) — P7.5 深度算法升级（拓扑关系引擎+走廊网络+DIM增强）

### 核心升级

#### 墙体-门窗拓扑关系引擎
- **精确匹配**：门 bbox 必须接触墙 bbox 的某条边（距离<50mm），取重叠最大的墙
- **投影约束**：门在墙边上的投影重叠度计算，confidence 随重叠比例变化
- **宿主墙注入**：门实体注入 `host_wall_id`/`host_wall_overlap` 属性

#### 走廊拓扑网络
- **连通图**：从走廊实体的相邻关系构建无向图，BFS 找连通分量
- **死胡同检测**：度=1 的走廊节点标记为死胡同，输出位置/宽度
- **宽度统计**：连通分量内走廊宽度平均值/最小值/最大值
- **走廊-门连通**：基于边缘距离（<200mm）建立走廊-门连接关系

#### 疏散路径分析
- BFS 从每个 room 搜索到最近 exit 的最短路径
- 路径长度 > 30m 标记为超标
- 输出路径节点序列

#### DIMENSION 注入增强（V2）
- **投影约束**：DIM 的 defpoint2/defpoint3 投影必须在实体 bbox 范围内
- **组合评分**：距离评分 + 投影重叠度加权，避免跨实体误匹配
- **单位转换优化**：双重检查机制，修复走廊宽度 1.096m 边界误报
- 完整提取 defpoint2/defpoint3/text_midpoint

#### 几何分类增强（V2）
- **短 LINE 归类**：50~500mm 细长 LINE → door
- **INSERT 块推断**：从块名（DOOR/WINDOW/STAIR）推断实体类型
- **TEXT 增强**：文本含"楼梯"/"防火"关键字归类
- **小面积闭合多边形**：狭长→door，近正方形→column
- **INSERT 属性提取**：block_name/insert_x/insert_y

#### 空间哈希加速
- 相邻关系构建从 O(n²) 降为近似 O(n)，1000+ 实体时显著加速
- CELL_SIZE=100mm 空间网格，只比较同一/相邻网格的实体

### 修复
- **EXIST 类误报**：修复 stair 实体因无 `exists` 属性被误判为缺失的问题
  - 真实图纸：实体存在（有属性）即 PASS
  - 合成图纸：META 实体的 `exists` 属性优先
- **走廊宽度注入**：平行线聚类结果 gap(mm) 正确转换为 m 并注入走廊 width
- **`ent.name` AttributeError**：修复 fire_door 防火等级推断中缺失的 name 属性
- **DIMENSION defpoint 完整提取**：drawing_parser._extract_dimensions 补齐 defpoint2/defpoint3

### 合成图纸生成器修复
- **ATTR-001 违规注入**：fire_rating 根据 violations 标记设为 0/1
- **EXIST-001 违规注入**：fail=True 时楼梯实体 `exists=False`
- 所有 register_entity 的 fire_rating/rating 动态化
