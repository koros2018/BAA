## 六、论文证据对财务判断的补充支撑（2026-06-09）

> 以下引用基于arXiv及已发表论文检索结果，为财务评估提供第三方学术证据。

### 6.1 启动资金争议

**原评估：** 需至少¥150万启动资金。

**论文证据：**
- **Fine-Tuning LLMs for Building Code QA**（Aqib et al., 2025, arXiv:2505.09686）：RAG+微调方案在**小模型**上即可达到82.7%的规范问答准确率，不需要大模型API调用。
- **LLM Attribution Analysis**（Shi et al., 2026, ICCCBE 2026）：小模型+领域微调可接近大模型水平。

**修正：年成本可进一步降低。**
- 用开源小模型（如Llama 3 8B/DeepSeek 7B）替代大模型API，本地部署
- 年API费用从¥2000-5000降至**接近¥0**
- 但需一台本地GPU机器（现有开发机即可，不需额外采购）
- **修正后年总成本：¥2700-5000**（仅ODA教育版+域名）

### 6.2 收入预测的学术支撑

**论文证据：**
- **Exploring Gen-AI in Building Research**（Wan et al., 2024, arXiv:2410.01096）：AI审图、规范合规检查被列为建筑AI最成熟的应用方向。
- **LLM-Driven Code Compliance in BIM**（Madireddy et al., 2025, arXiv:2506.12814）：AI合规检查可减少60%手动审查时间。

**修正：Y1收入¥30万仍然可行。** 学术证据支持产品价值主张，但付费意愿仍需要通过实际客户接触验证。

### 6.3 融资策略修正

**原建议：** MVP完成后开始接触天使投资人。

**论文证据支撑的修正：**
- 一人公司+零成本模式下，BAA不需要外部融资即可运营
- 但如有扩张计划（如雇佣员工、申请行业认证），可考虑融资
- 建议：**先自负盈亏验证PMF，再决定是否融资**。而不是先融资再验证。

## 七、核心建议

1. **启动资金≈¥0。** 用开源小模型替代大模型API后，年运营成本仅¥2700-5000。不需要¥150万。
2. **Y1收入目标¥30万。** 学术证据支持产品价值，但付费意愿仍需实际验证。
3. **现金流管理第一。** 年成本不超过¥5000，即使Y1零收入也毫无压力。
4. **年付优先。** 给年付客户15-20%折扣，改善现金流。
5. **先验证PMF，再决定是否融资。** 一人公司模式下融资不是必需品。
