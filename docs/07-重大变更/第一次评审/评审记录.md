## 会议二：技术评审会

### 2.1 技术架构

```
用户 → Windows风格Web前端 → FastAPI后端 → AI Agent编排引擎
                                            │
                              ┌─────────────┼─────────────┐
                              ▼             ▼             ▼
                         SubAgent-001   SubAgent-002   知识库引擎
                         (合规审查)     (生成修正)    (规范+图集+过程)
                              │             │             │
                              ▼             ▼             ▼
                         ODA/ezdxf     参数化模板    PostgreSQL
                         规则引擎      CAD引擎       ChromaDB
                                         LLM          MinIO
```

### 2.2 技术选型

| 决策点 | 推荐 | 理由 |
|--------|------|------|
| CAD引擎 | 先ezdxf过渡→ODA商业 | 零成本启动，收入确认后采购ODA |
| LLM | 先云端API→混合 | 零运维，快速启动 |
| 前端 | 先Web→Electron桌面 | Web快速验证，Electron做正式版 |
| Agent框架 | 自研轻量工作流引擎 | LangChain太重，自研更可控 |
| 数据库 | PostgreSQL | JSONB支持好，向量扩展成熟 |
| 向量库 | ChromaDB | 最轻量，适合初期 |
| 任务队列 | Celery | 生态最成熟 |

### 2.3 核心技术难点

| 难点 | 方案 |
|------|------|
| 图纸解构几何保真度 | ODA SDK（商业）或 libredwg-web（WASM） |
| 规范条目可执行化 | 人工录入核心条款 + LLM辅助解析 |
| 闭环修正稳定性 | 迭代上限20次 + 回溯机制 + 修正评分函数 |
| LLM幻觉 | 规则引擎为主 + LLM为辅 + 每条判定引用具体条款 |

### 2.4 技术评审结论

**✅ 技术方案可行，分阶段实施。** 核心风险可控。
