**论文#14：Parsing Line Segments of Floor Plan Images Using Graph Neural Networks**
- **作者：** Chen & Pan (2023)
- **来源：** arXiv:2303.03851
- **核心发现：** 提出了GNN-based Line Segment Parser (GLSP)，用junction heatmap预测线段端点，GNN提取线段及分类。发布了**Large-scale Residential Floor Plan (LRFP)数据集，共271,035张平面图**，标注包含比例信息、房间类别/轮廓、门窗墙线段端点位置。数据增强方法使数据集可适应多国不同绘图风格。
- **对BAA初步设计的支撑：** ✅ **合成数据方案确认为可行。** 27万张平面图数据集的规模证明"足够大+数据增强=可适应多国风格"的路线在学术界已被验证。BAA可参考此数据增强策略构建自己的训练数据集。
- **详细设计中可用：** 该论文的GNN线端检测方法可用于BAA语义识别层的线端解析模块。

**论文#15：Component Segmentation of Engineering Drawings Using Graph Convolutional Networks**
- **作者：** Zhang, Joseph, Yin et al. (2022)
- **来源：** arXiv (Dec 2022)
- **核心发现：** 用图卷积网络（GCN）对工程图纸进行组件分割。将工程图纸的几何元素（线、弧、文字）构建为图结构，节点为图元，边为空间关系。在多种复杂工程图纸上验证了GCN方法在识别结构化组件方面的优势。
- **对BAA初步设计的支撑：** ✅ **为语义识别层提供GCN备选方案。** 如果未来BAA需要处理更复杂的结构化图纸（如设备布置图、管线图），GCN方案值得参考。

**论文#16：Explainability as a Compliance Requirement**
- **作者：** Shah, Hussein, Barcomb, Moshirpour (2025)
- **来源：** arXiv (Jul 2025)
- **核心发现：** 研究受监管行业对AI工具的可解释性要求。结论是：在受监管行业中，AI工具生成的每一个设计工件必须可追溯、可审计、可复核。归因分析（attribution）是合规的最低要求。
- **对BAA初步设计的支撑：** ✅ **可解释性BAA核心差异化的关键学术支撑。** 审图是受监管行业（施工图审查制度），"可追溯的审查报告"不是加分项，是合规底线。坚决不做黑箱输出。

**论文#11：RADIANT-LLM——安全关键工程中的Agentic RAG框架**
- **作者：** Ndum, Tao, Ford, Yim, Liu (2026)
- **来源：** arXiv:2604.22755
- **核心发现：**
  - 针对核工程安全关键领域，设计多模态RAG框架
  - **本地优先、模型无关**架构 + 多模态文档处理管线 + 结构化元数据知识库
  - 支持页码级和图级别的检索
  - **Agentic层**协调领域特定工具，强制引用回溯（citation-backed responses）
  - 人工确认回路（human-in-the-loop validation）降低幻觉风险
  - 在Used Nuclear Fuel Storage Facility设计基准上：**Context Precision和Visual Recall在85-98%区间**，幻觉率显著低于通用部署
  - 结论：安全关键工程中，必须使用本地控制的多模态RAG框架才能达到审计要求
- **对BAA初步设计的支撑：** ✅ **RAG架构选择的决定性证据。**
  - RADIANT-LLM与BAA的场景高度相似：都是安全关键工程（核安全≈建筑防火安全）
  - 其Agentic RAG + 本地优先 + 人工确认回路的架构，可直接作为BAA核心推理层的参考架构
  - Context Precision 85-98%表明：领域特定RAG在安全关键场景下可以达到可用的准确率
- **详细设计中可用：** Agentic层的工具协调设计、人工确认回路的具体实现。

**论文#17：DiffPlanner——直接矢量平面图生成**
- **作者：** Wang & Pajarola (2025)
- **来源：** arXiv:2508.13738 / IEEE TVCG 2025
- **核心发现：**
  - 提出完全在矢量空间中进行的平面图生成框架
  - **Transformer + 条件扩散模型**，直接输出矢量格式，避免"栅格化→生成→反矢量"的信息丢失
  - 引入对齐机制（alignment mechanism），使模型优化轨迹与设计师的迭代设计过程一致
  - 支持用户可控生成
  - 在平面图和气泡图生成任务上超越现有SOTA方法
- **对BAA初步设计的支撑：** 🟡 **中期参考——V2.0意图驱动设计的潜在技术路线。**
  - 当前BAA只做合规审查（判别式），不做生成式
  - 如果未来V2.0要做"意图→图纸"生成，DiffPlanner的完全矢量空间方法是避免信息丢失的关键
  - 其Transformer+扩散模型的选择可作为生成式Agent的技术储备

## 第二阶段：详细设计（实现级）

### DD-1：规范原子函数库详细设计

**设计目标：** 构建BAA规范判定原子函数库，参考论文#1的66个原子函数体系。

**论文支撑细节：**

**论文#1 (LLM-FuncMapper)：** 66个原子函数分为以下类别（基于论文摘要推断）：

| 函数类别 | 示例 | 对应的BAA规范条款 |
|---------|------|-----------------|
| 尺寸判定类 | `check_dimension(entity, prop, op, threshold)` | 疏散宽度、房间面积、净高 |
| 距离判定类 | `check_distance(entity_a, entity_b, op, threshold)` | 疏散距离、防火间距 |
| 存在性判定类 | `check_existence(layer, entity_type, required)` | 防火门、疏散指示标志 |
| 计数判定类 | `check_count(layer, entity_type, op, threshold)` | 安全出口数量、消防电梯数量 |
| 属性判定类 | `check_attribute(entity, attr, op, value)` | 防火门等级、材料燃烧等级 |
| 关系判定类 | `check_relation(entity_a, rel, entity_b)` | 楼梯间通往、防火分区连通 |
| 复合判定类 | `combine(checks, logic=AND/OR/NOT)` | 多种条件组合的条款 |

**BAA原子函数签名规范（建议）：**

```python
class AtomicFunction:
    """
    原子函数基类
    每个原子函数封装：
    1. 几何参数提取逻辑（从结构化图纸数据中提取所需参数）
    2. 规范阈值判定（与阈值比较）
    3. 违规报告生成（附带归因分析）
    """
    id: str          # 唯一标识，如 "AF-DIM-001"
    name: str        # 中文名称，如 "疏散宽度判定"
    category: str    # 类别，如 "dimension" / "distance" / "existence" / "count" / "attribute" / "relation"
    params: dict     # 参数定义

    def extract(self, drawing_data: DrawingData) -> ExtractedParam:
        """从图纸数据中提取判定所需参数"""
        pass

    def judge(self, param: ExtractedParam) -> Judgement:
        """执行规范阈值判定，返回判定结果"""
        pass

    def explain(self, judgement: Judgement) -> Explanation:
        """生成可解释的违规说明"""
        pass
```

**原子函数可组合：**
```python
# 复合函数示例：疏散楼梯是否合规
def check_evacuation_stair():
    return AND(
        check_stair_width(...),   # AF-DIM-003: 楼梯净宽≥1.2m
        check_exit_count(...),     # AF-COUNT-001: 安全出口≥2个
        check_exit_distance(...),  # AF-DIST-005: 疏散距离≤30m
        check_door_type(...)       # AF-ATTR-002: 防火门等级甲级
    )
```

```json
  "explanation": "模型重点关注了ST-03（疏散楼梯，注意力权重0.87），检测到净宽1.05m<1.2m。"

```
